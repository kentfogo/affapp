# AffApp - Requirements & Success Criteria

## Project Overview
An affirmation app built with Flutter and Firebase that helps users hear personalized affirmations at intervals during movement sessions.

## Goals and Success Criteria

### Primary Goals
- Users can create an account, complete onboarding questions, and pick 5â€“10 affirmations from a 400-item bank or add recorded ones.
- Session screen supports time and distance intervals; affirmation playback occurs at each interval; current affirmation displayed.
- Long-press stop ends session; a summary screen records the session and shows positive reinforcement; user returns to the start screen.

### How will users know this solved their problem?
By the end of their session, they have heard the type of affirmations that they needed to hear on repeat at the desired interval and it repeats in their brain.

### What does "working well" look like in real usage?
They open the app, click the Start Movement button, hear their affirmations get repeated to them in their desired interval, get positive reinforcement afterward and then close the app.

## Specific Outcomes - User Flows

### New Users
1. Open the app
2. Complete the survey to determine which type of affirmations are right for them
3. Choose whether to record their own affirmations or use affirmations generated by the app
4. Choose whether to hear their own voice or voice provided by the app
5. Choose 5-10 favorite affirmations
6. Start Movement button
7. Hear your affirmations in your desired interval according to the previously selected preferences
8. End the session
9. Positive reinforcement screen
10. Done

### Active Users
1. Open the App
2. Hit the start movement button
3. Hear your affirmations in your desired interval according to the previously selected preferences
4. End the session
5. Positive reinforcement screen
6. Done

### All Experiences
- Custom Affirmations screened for negativity
- Smooth Experience
- Security Best Principles are enacted

## Technical Requirements

### Tech Stack
- **Frontend**: Flutter
- **Backend**: Firebase
  - Authentication
  - Firestore Database
  - Cloud Storage (for recorded affirmations)
  - Cloud Functions (for affirmation screening)

### Core Features

#### Authentication & Onboarding
- User account creation
- Onboarding survey/questionnaire
- Profile setup

#### Affirmation Management
- 400-item affirmation bank
- User can select 5-10 affirmations
- Option to record custom affirmations
- Custom affirmations must be screened for negativity
- Voice selection (user's voice or app-provided voice)

#### Session Management
- Start Movement button
- Time interval support
- Distance interval support
- Affirmation playback at intervals
- Current affirmation display
- Long-press to stop session
- Session recording/tracking

#### Post-Session
- Summary screen
- Positive reinforcement
- Session history

### Security Requirements
- Secure authentication
- Input validation
- Data sanitization
- Secure storage of user data
- Negativity screening for custom affirmations

## Success Metrics
- Users complete onboarding successfully
- Users can start and complete sessions
- Affirmations play at correct intervals
- Custom affirmations are properly screened
- Positive user experience with smooth interactions
